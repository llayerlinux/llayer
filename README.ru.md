# lastlayer (llayer)

Динамический интерфейсный слой — новая абстракция над окружением рабочего стола, позволяющая управлять средой на нескольких уровнях и мгновенно преобразовывать её.

## Установка

```bash
git clone https://github.com/llayerlinux/lastlayer.git
cd lastlayer
./install.sh
```

> **Примечание**  
> Все необходимые зависимости устанавливаются автоматически скриптом `install.sh`: swww, yad, webkit2gtk.

## Возможности

### Управление rice'ами

- Переключение rice одним кликом. Среднее время применения — ~2 секунды (и продолжает улучшаться).
- Установка rice из интернета или локальный импорт из файловой системы.
- Возможность делиться rice: загружайте свои rice (dotfiles) на сервер для публичного доступа.
- Редактирование/удаление загруженного вами rice через GUI.
- Открытие репозитория конфигурации, просмотр поддерживаемых дистрибутивов и синхронизация основной информации через Git.

### Конфигурации и плагины

- Управление конфигурациями через GUI.
- Управление плагинами через GUI:
  - добавление репозиториев плагинов
  - добавление пользовательских параметров к плагину
  - поддержка разных типов параметров (например, выбор цвета)

### Инструменты стабильности

- Исправление проблемных временных состояний поддерживаемых тайлинговых оконных менеджеров.
- Сохранение и восстановление внешних состояний окружения.

### Безопасность и изоляция

- Проверка скриптов rice на безопасность перед выполнением + пользовательские правила безопасности.
- Двухуровневая изоляция зависимостей rice: стандартная + управляемая префиксная система (beta).

Если в скрипте rice обнаружены символические ссылки, они получают приоритет, а стандартный механизм отключается.

### UX и качество жизни

- Настройка анимации переключения rice (сейчас используется `swww`):
  - тип анимации
  - FPS
  - длительность
  - угол волны
- Дополнительные опции:
  - автоприменение rice после загрузки (опционально)
  - оставлять или закрывать окно списка rice после выбора
  - включать логирование времени применения/установки
- Многоязычная поддержка: сейчас поддерживается 4 языка.
- Параметры lastlayer:
  - переключение темы интерфейса
  - базовое управление звуком

## Стандарт rice

### Карточка rice (текущая)

Карточка rice — одно из ключевых всплывающих окон; в будущих версиях она будет расширяться.

Rice должен включать:

- Предпросмотр (512x512 или 1024x1024)
- Ссылка на репозиторий
- Автор (аватар автоматически парсится из Git)
- Автор адаптации (опционально)
- Теги (опционально)
- Категория функциональности (опционально; допускается несколько)
- Поддерживаемые дистрибутивы (опционально; если не указано, lastlayer генерирует список автоматически на основе скрипта)

Запланировано:

- Автоматическая конвертация между дистрибутивами и тайлинговыми оконными менеджерами
- Система рейтинга (подробнее — в roadmap)

## Адаптация существующего rice / создание нового

В первых версиях программе временно требуется предсказуемая структура rice.

### Минимально приемлемая структура (Hyprland)

```text
RICE_NAME/
├── preview.png
├── wallpaper.png
├── hyprland.conf
├── hyprland/
│   ├── lastlayer.conf
└── start-scripts/
    ├── install_theme_apps.sh (опционально)
    └── set_after_install_actions.sh
├── config/ (опционально)
```

### Рекомендуемая структура

```text
RICE_NAME/
├── preview.png
├── wallpaper.png
├── hyprland.conf
├── hyprland/
│   ├── env.conf
│   ├── execs.conf
│   ├── general.conf
│   ├── rules.conf
│   ├── colors.conf
│   ├── keybinds.conf
│   └── lastlayer.conf
└── start-scripts/
    ├── install_theme_apps.sh (опционально)
    └── set_after_install_actions.sh
├── config/ (опционально)
```

### Значение файлов

- `preview.png` — изображение, показываемое пользователю в списке rice
- `wallpaper.png` — обои рабочего стола по умолчанию
- `hyprland.conf` — ссылки на файлы конфигурации Hyprland (возможны и прямые параметры без ссылок). Файлы, относящиеся к этой конфигурации, должны располагаться в `hyprland/`.
- `lastlayer.conf` — конфигурация lastlayer, которая обеспечивает консистентное переключение, скрывает вспомогательные терминалы во время переключения и задаёт необходимые атрибуты видимости для окна программы

### start-scripts/

- `install_theme_apps.sh` (опционально)  
  Устанавливает сопутствующие приложения. Запускается только при первой установке rice.
- `set_after_install_actions.sh`  
  Запускает сопутствующие приложения. Выполняется каждый раз при применении rice.

### config/ (опционально)

Директория с конфигурационными файлами сопутствующих приложений.

> **Совет**  
> Примеры можно посмотреть в предустановленных rice в разделе **Network** lastlayer (или в стартовом архиве выше). Они следуют распространённому подходу: разбивать конфиг Hyprland на несколько файлов и ссылаться на них из `hyprland.conf`.

Примечания о совместимости и скриптах (текущий подход):

- Пока что lastlayer ожидает предсказуемую структуру rice. В будущих версиях можно будет импортировать legacy-rice без реструктурирования (rice, которые уже следуют этому layout, останутся обратно совместимыми).
- Скрипты rice устанавливают и запускают сопутствующие приложения, чтобы все действия были прозрачны и легко проверялись.
- lastlayer распознаёт потенциально опасные команды и не запускает их без явного согласия пользователя.

## Roadmap

- Кодовая база открыта, архитектура подготовлена под будущие возможности
- Интеграция метаданных рейтинга с Reddit и ссылки на YouTube-демо в карточки rice
- Переключение rice на уровне всего PC-флоу (GRUB, rEFInd, экран входа)
- Поддержка других популярных оконных менеджеров (rice, конфиги, плагины)
- Конвертация/маппинг rice между оконными менеджерами (модуль эквивалентных параметров) и конвертация install/apply-пакетов между дистрибутивами
- Модуль обратной совместимости для legacy-rice
- Улучшения UI: drag and drop, фильтры по тегам, пагинация для элементов Network
- Экспериментально: переключение по текущему режиму работы или по контексту активного окна
- AI-генерация окружений/rice (на основе промпта и контекста)
