# ![icon](https://github.com/user-attachments/assets/27330896-e1fd-47d2-83cb-463c46a73475) lastlayer (llayer)

Динамический интерфейсный слой — новая абстракция поверх окружения рабочего стола, которая позволяет управлять окружением на нескольких уровнях и мгновенно его преобразовывать.

## Установка

```bash
git clone https://github.com/llayerlinux/lastlayer.git
cd lastlayer
./install.sh
```

Примечание

Все необходимые зависимости устанавливаются автоматически скриптом `install.sh`: swww, yad, webkit2gtk.

## Возможности
### Управление rices

- Переключение rice в один клик. Среднее время применения ~2 секунды (и продолжает улучшаться).
- Установка rices из интернета или локальный импорт из файловой системы.
- Обмен rices. Загружайте свои rices (dotfiles) на сервер для публичного доступа.
- Редактирование/удаление загруженного rice из GUI.
- Открытие репозитория конфигурации, просмотр поддерживаемых дистрибутивов и синхронизация основной информации через Git.

### Конфиги и плагины

- Управление конфигурациями через GUI.
- Управление плагинами через GUI:
  - добавление репозиториев плагинов
  - добавление пользовательских параметров в плагин
  - поддержка разных типов параметров (например, выбор цвета)

### Инструменты стабильности

- Исправление проблемных временных состояний поддерживаемых тайлинговых оконных менеджеров.
- Сохранение и восстановление внешних состояний окружения.

### Безопасность и изоляция

- Проверка безопасности скриптов rice перед выполнением + пользовательские правила безопасности.
- Двухуровневая изоляция зависимостей rice: стандартная + система контролируемых префиксов (beta).

Если в скрипте rice обнаружены симлинки, они получают приоритет, а стандартный механизм отключается.

### UX и удобство

- Настройка анимации переключения rice (сейчас используется `swww`):
  - тип анимации
  - FPS
  - длительность
  - угол волны
- Дополнительные опции:
  - авто-применение rice после загрузки (опционально)
  - оставить или закрыть окно списка rices после выбора
  - включить логирование времени применения/установки
- Мультиязычность: сейчас поддерживается 4 языка.
- Параметры lastlayer:
  - переключение темы интерфейса
  - базовое управление звуком

### Карточка rice (текущая)

Карточка rice — одно из ключевых всплывающих окон и будет расширяться в будущих версиях.

Rice должен включать:

- Превью (512x512 или 1024x1024)
- Ссылка на репозиторий
- Автор (аватар автоматически парсится из Git)
- Автор адаптации (опционально)
- Теги (опционально)
- Категория функциональности (опционально; допускается несколько)
- Поддерживаемые дистрибутивы (опционально; если не указано, lastlayer генерирует список автоматически на основе скрипта)

Планируется
- Автоматическая конвертация между дистрибутивами и тайлинговыми оконными менеджерами
- Система рейтинга (подробнее в дорожной карте)

## Адаптация существующего rice / создание нового

В первых версиях программа временно требует предсказуемую структуру rice.

### Минимально допустимая структура (Hyprland)

```text
RICE_NAME/
├── preview.png
├── wallpaper.png
├── hyprland.conf
├── hyprland/
│   ├── lastlayer.conf
└── start-scripts/
    ├── install_theme_apps.sh (optional)
    └── set_after_install_actions.sh
├── config/ (optional)
```

### Рекомендуемая структура

```text
RICE_NAME/
├── preview.png
├── wallpaper.png
├── hyprland.conf
├── hyprland/
│   ├── env.conf
│   ├── execs.conf
│   ├── general.conf
│   ├── rules.conf
│   ├── colors.conf
│   ├── keybinds.conf
│   └── lastlayer.conf
└── start-scripts/
    ├── install_theme_apps.sh (optional)
    └── set_after_install_actions.sh
├── config/ (optional)
```

### Значение файлов

#### hyprland/

- `lastlayer.conf`: специальный файл с оверрайдами конфигурации lastlayer. lastlayer применяет его после применения основной конфигурации Hyprland.

#### start-scripts/

- `install_theme_apps.sh` (optional)  
  Устанавливает сопутствующие приложения. Запускается только при первой установке rice.
- `set_after_install_actions.sh`  
  Запускает сопутствующие приложения. Выполняется каждый раз при применении rice.

#### config/ (optional)

Директория с конфигурационными файлами сопутствующих приложений.

Совет

Примеры можно посмотреть в предустановленных rices в разделе lastlayer Network (или в стартовом архиве выше). Они следуют распространённому подходу: разносить конфиг Hyprland по нескольким файлам и подключать их из `hyprland.conf`.

Заметки о совместимости и скриптах (текущий подход)
- Пока что lastlayer ожидает предсказуемую структуру rice. В будущих версиях вы сможете импортировать legacy-rices без реструктуризации (rices, которые уже следуют этому макету, сохранят обратную совместимость).
- Скрипты rice устанавливают и запускают сопутствующие приложения, чтобы все действия были прозрачными и легко проверяемыми.
- lastlayer распознаёт потенциально опасные команды и не выполняет их без явного согласия пользователя.
